# Phase4 断板测信号清单

## 当前默认入口

- 当前 `User/Test/Inc/test.h` 中 `TEST_ACTIVE_PHASE` 默认配置为 `TEST_PHASE_1_ID`
- 这表示上电后默认进入 `Phase1`，用于纯输出链路测量
- 当 `Phase1` 验证通过后，把 `TEST_ACTIVE_PHASE` 改成 `TEST_PHASE_4_ID`，再做伪 Hall 注入测试

## 场景 A: Phase1 纯输出链路测量

- 控制板单独上电
- 驱动板完全断开
- 母线不上电
- 使用 `Phase1`，不依赖 Hall

### 测量点

- `PC13`: 驱动使能
- `PA8/PA9/PA10`: 三相上桥 PWM
- `PB13/PB14/PB15`: 三相下桥 GPIO

### 预期结果

- `Step0/Step5`: `PC13` 低，`PA8/PA9/PA10` 无 PWM，`PB13/PB14/PB15` 全低
- `Step1`: `PC13` 高，其他输出保持安全态
- `Step2`: `PA8/PA9/PA10` 三路约 `10kHz` PWM
- `Step3`: 只有 `U` 相有 PWM，其余两相无 PWM
- `Step4`: `PB13/PB14/PB15` 输出代码定义的下桥组合

## 场景 B: Phase4 伪 Hall 注入测量

- 控制板单独上电
- 驱动板完全断开
- 母线不上电
- 把 `TEST_ACTIVE_PHASE` 改成 `TEST_PHASE_4_ID`
- 用控制板本机 `3.3V/GND` 人工给 `PA0/PA1/PA2` 注入 Hall 电平

### 约束

- Hall 输入只允许 `0V` 或 `3.3V`
- 不允许外部 `5V`
- 只使用有效状态: `001 / 101 / 100 / 110 / 010 / 011`
- 每次只切换一位 Hall，避免制造非法跳变

### 初始建议

- 先固定 `Hall=001`
- 按 `KEY0` 启动，确认串口进入 `Run=1`
- 此时测量对应动作输出

### 静态动作判据

- 若串口显示 `Act=W+V-`
- 则应看到 `PC13` 高
- `PA10` 有 PWM
- `PA8/PA9` 无 PWM
- `PB14` 为高
- `PB13/PB15` 为低

### 手动 Hall 序列

- 验证 `CCW`: `001 -> 101 -> 100 -> 110 -> 010 -> 011 -> 001`
- 验证 `CW`: `001 -> 011 -> 010 -> 110 -> 100 -> 101 -> 001`

### 每次切换后观察

- 串口 `Hall=` 是否与注入状态一致
- 串口 `Edge/Valid` 是否递增
- 串口 `Act=` 是否同步变化
- 对应 PWM 和下桥输出是否一起变化

## 结论判据

- `Phase1` 通过: 控制板输出链路正确
- `Phase4` 伪 Hall 跟随正确: Hall 采样和换相表链路正确
- 两者都通过后，再恢复驱动板连接，进入低压限流实机测试
